message MessagingComponentConsumerSettings

Defines configuration settings for a specific consumer within a messaging component. Each consumer represents a logical queue handler that processes messages from subscribed subjects.

Field Type Description
1 consumerId string

The unique UUID identifier for this consumer instance. Must be unique across all consumers of the same component. @gqltype: UUID
2 maxInFlightMessages int32

The maximum number of unacknowledged messages this consumer can have. Values >1 enable concurrent processing but may lose order. Value 1 guarantees ordered processing. @example: 1 @example: 10 @example: 100
3 subjects repeated string

The message subjects this consumer subscribes to. At least one subject is required. Changes apply to all instances of the component. @example: ["events.public.acquisition.jobdone", "events.public.something.else"] @example: ["events.custom.sapmessage"]

message MessagingConsumerClient

Defines messages sent from a consumer client to the messaging server in a bidirectional stream. Consumers use this to establish a connection, acknowledge processed messages, or reject failed messages.

Field Type Description
oneof kind
1 setup io.clbs.openhes.models.messaging.MessagingConsumerSetup

Setup message to initialize the consumer connection. Must be the first message sent. Subsequent setup messages are rejected.
2 ack google.protobuf.StringValue

Acknowledgement confirming successful processing of a message. The message is removed from the queue.
3 nak google.protobuf.StringValue

Negative acknowledgement indicating processing failure. The message is requeued for redelivery.
